Details File Mapping Column

Description

The CDA type which is specified as the mapping for an SCS data component in a logical model "Details" XML file in one of its <Mapping> elements. This information is used by the LMI to create the UML structure which is deduced from the type information in the context of the SCS Tree, and the CDA structure that was cteated for the parents of this SCS component.

The <Mapping> element which this cell refers to contains an XPath expression that has a CDA Document path (and hence an iferrable CDA type) which gives the type of the equivalent CDA element for the SCS component, and the rest of the XPath expression forms a constraint, which often fixes the value. See Dts: Fixed Value.

Correlates With

This value is used by the Logical Model Importer algorithm to choose the right type for the CDA equivalent of a data component, in its SCS context, and so it forms part of a trace 4 tuple: (SCS: Data Compoenent, Dts: Mapping, UML: Type, UML: Element).

Cells Navigate To

The XML Details file which is the source of the value.
