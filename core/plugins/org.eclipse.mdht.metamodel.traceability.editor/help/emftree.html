The EMF Tree provides a EMF-native view onto the traceability model. All information in the traceability tables originate from the traceability model and the domain elements it links to.

Tree item types

Most of the items in the EMF Tree are Trace instances which map source elements to target elements, e.g. a requirement to a data component. However also items for error message can be contained in the EMF Tree. Error message are graded into hints, warnings and errors, associated with an appropriate icon. Also, there are Category nodes for structuring traces.

Tree structure

There are category items to group requirement, SCS and vocabulary traces/errors. Traces/errors for the LMI import are not put into a category but placed in the outermost level, to ease the access to them.

Interactivity

Traces/errors have commonly child nodes which are the subjects of an trace/error. By double-clicking them, they can be opened and navigated in their (default Eclipse) editor.
