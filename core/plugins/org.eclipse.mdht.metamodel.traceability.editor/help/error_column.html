Error Column

Description

This is a special column that aggregates any errors that are detected in correlations or mappings between the model sources. These may be LMI Import errors for SCS elements that cannot be inserted correctly into the UML CDA structure based on their re-usable mappings, or they may be failed correlations between columns that are meant to match.

Key Status

Primary Key

Technology Space

EMF Trace metamodel instance

Cardinality

1

Correlates With

SCS: Data Component AND/OR UML: Element when an element is created by LMI, but is not complete or consistent with mapping rules.

Cell Navigation Links To

When no UML element was created: to the trace file, and trace instance where a SCS Data Component could not be used to create an MDHT CDA UML element because a re-usable mapping could not be found.

OR when an MDHT CDA model element could be created, but is incomplete or inconsistent with the CDA mapping rules, the celss navigates to the MDHT model element in the UML Table editor.
